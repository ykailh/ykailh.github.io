Spring入门之Spring简介

2020-09-20
Spring

简单了解Spring

1.什么是Spring

​ Spring是一个开源框架，它可以通过简单的配置实现以前EJB容器（不了解可以跳过）的功能，它最核心的两个思想一个是IOC（控制反转），一个是AOP（面向切面编程）。

对于很多新的初学者，可能是第一次接触到框架的概念，简单来说框架就是一个半成品的组件，它对一些底层的代码进行了封装，使之变得简单易用。相当于框架已经帮助你完成了许多基础工作，你只需要集中精力完成系统的业务逻辑。

2.Spring的优良特性

  • 非侵入式：基于Spring开发的应用中的对象可以不依赖于Spring的API。

  • 控制反转：IOC——Inversion of Control，指的是将对象的创建权交给 Spring 去创建。使用 Spring 之前，对象的创建都是由我们自己在代码中new创建。而使用 Spring 之后。对象的创建都是给了 Spring 框架。

  • 依赖注入：DI——Dependency Injection，是指依赖的对象不需要手动调用 setXX 方法去设置，而是通过配置赋值，它是IOC思想最经典的实现方式之一。

  • 面向切面编程：AOP——Aspect Oriented Programming，它关注的是横跨多个点的业务逻辑以外的功能，是对面向对象编程（OOP）的补充。

  • 容器：Spring 是一个容器，因为它包含并且管理应用对象的生命周期。

  • 组件化：Spring 实现了使用简单的组件配置组合成一个复杂的应用。在 Spring 中可以使用XML和Java注解组合这些对象。

  • 一站式：在 IOC 和 AOP 的基础上可以整合各种企业应用的开源框架和优秀的第三方类库（实际上 Spring 自身也提供了表述层的 SpringMVC 和持久层的 Spring JDBC）

这些特性可能只看概念可能体会不到这些好处，以后在学习和实际使用中加深理解。

Spring的体系结构

​ Spring 有可能成为所有企业应用程序的一站式服务点，然而，Spring 是模块化的，允许你挑选和选择适用于你的模块，不必要把剩余部分也引入。

1.核心容器

核心容器由 spring-core，spring-beans，spring-context，spring-context-support和spring-expression（SpEL，Spring 表达式语言，Spring Expression Language）等模块组成，它们的细节如下：

  • spring-core 模块提供了框架的基本组成部分，包括 IoC 和依赖注入功能。
  • spring-beans 模块提供 BeanFactory，工厂模式的微妙实现，它移除了编码式单例的需要，并且可以把配置和依赖从实际编码逻辑中解耦。
  • context 模块建立在由 core和 beans 模块的基础上建立起来的，它以一种类似于 JNDI 注册的方式访问对象。Context 模块继承自 Bean 模块，并且添加了国际化（比如，使用资源束）、事件传播、资源加载和透明地创建上下文（比如，通过 Servelet 容器）等功能。Context 模块也支持 Java EE 的功能，比如 EJB、JMX 和远程调用等。ApplicationContext 接口是 Context 模块的焦点。spring-context-support 提供了对第三方库集成到 Spring 上下文的支持，比如缓存（EhCache, Guava, JCache）、邮件（JavaMail）、调度（CommonJ, Quartz）、模板引擎（FreeMarker, JasperReports, Velocity）等。
  • spring-expression 模块提供了强大的表达式语言，用于在运行时查询和操作对象图。它是 JSP2.1 规范中定义的统一表达式语言的扩展，支持 set 和 get 属性值、属性赋值、方法调用、访问数组集合及索引的内容、逻辑算术运算、命名变量、通过名字从 Spring IoC 容器检索对象，还支持列表的投影、选择以及聚合等。

2.数据访问/集成

  • JDBC 模块提供了 JDBC 抽象层，它消除了冗长的 JDBC 编码和对数据库供应商特定错误代码的解析。
  • ORM 模块提供了对流行的对象关系映射 API 的集成，包括 JPA、JDO 和 Hibernate 等。通过此模块可以让这些 ORM 框架和 spring的其它功能整合，比如前面提及的事务管理。
  • OXM 模块提供了对 OXM 实现的支持，比如 JAXB、Castor、XML Beans、JiBX、XStream 等。
  • JMS 模块包含生产（produce）和消费（consume）消息的功能。从 Spring 4.1 开始，集成了 spring-messaging 模块。
  • 事务模块为实现特殊接口类及所有的 POJO 支持编程式和声明式事务管理。（注：编程式事务需要自己写 beginTransaction()、commit()、rollback() 等事务管理方法，声明式事务是通过注解或配置由 spring 自动处理，编程式事务粒度更细）

3.Web

  • Web 模块提供面向 web 的基本功能和面向 web 的应用上下文，比如多部分（multipart）文件上传功能、使用 Servlet 监听器初始化 IoC 容器等。它还包括 HTTP 客户端以及 Spring 远程调用中与 web 相关的部分。
  • Web-MVC 模块为 web 应用提供了模型视图控制（MVC）和 REST Web服务的实现。Spring 的 MVC 框架可以使领域模型代码和 web 表单完全地分离，且可以与 Spring 框架的其它所有功能进行集成。
  • Web-Socket 模块为 WebSocket-based 提供了支持，而且在 web 应用程序中提供了客户端和服务器端之间通信的两种方式。
  • Web-Portlet 模块提供了用于 Portlet 环境的 MVC 实现，并反映了 spring-webmvc 模块的功能。

4.其他

  • AOP 模块提供了面向方面（切面）的编程实现，允许你定义方法拦截器和切入点对代码进行干净地解耦，从而使实现功能的代码彻底的解耦出来。使用源码级的元数据，可以用类似于.Net属性的方式合并行为信息到代码中。
  • Aspects 模块提供了与 AspectJ 的集成，这是一个功能强大且成熟的面向切面编程（AOP）框架。
  • Instrumentation 模块在一定的应用服务器中提供了类 instrumentation 的支持和类加载器的实现。
  • Messaging 模块为 STOMP 提供了支持作为在应用程序中 WebSocket 子协议的使用。它也支持一个注解编程模型，它是为了选路和处理来自 WebSocket 客户端的 STOMP 信息。

什么是泛型

​ 泛型可以按照字面简单理解为泛化的类型。

类比到生活中，就是生产瓶子的厂商不给瓶子贴上包装，而是预留了包装的位置。等到灌装前再根据灌装的内容贴上不同的包装。

​ 即将类型明确的操作推迟到创建对象或者调用方法时，同时把类型当中参数一样传递。

Java基础之集合

2020-08-11
Java

​ 本章内容主要是对Java中的集合的简单了解以及使用，在深入学习数据结构再进行学习和总结。

集合简介

1.什么是集合

​ 集合是用来存放数据的容器，主要有set(集)、list(列表)、map(映射)三种。

​ 与集合类似的还有之前接触到的数组，集合与数组的主要区别如下：

  • 数组的长度是固定的，集合的长度是可变的。
  • 数组可以存放基本数据类型也可以存放引用数据类型，集合只能存放引用数据类型。
  • 数据的读取速度快于集合，但占用连续的内存空间。

2.简单了解几个数据结构

​ 每个集合的底层存储方式可能都不同，导致了每个集合都有自己的性能特性，在实际使用中根据需求选择不同的集合。为了意识到这一点，简单介绍一些数据结构。

  • 数组

    ​ 数组的底层存储方式是根据数组的长度开辟一段连续的内存空间，将这段内存的首地址存放到以数组名为变量名的变量中，访问数据时根据首地址以及数组下标（偏移量）计算出数据所在的地址，访问数据。

  • 链表

    ​ 链表的底层存储方式是，将数据存放到不要求内存连续的空间中，每个内存空间称为一个信息节点。信息节点除了包含存储的数据外还包含下一个节点的地址值，访问数据时需要从第一个节点开始，读取节点中的地址值，在根据这个地址值去查找下一个节点，以此类推直到查询到的数据为止。这样的链表称为单向链表。

    若在节点中同时存放上一个节点的地址，则从两侧均可访问链表，这样的链表称为双向链表。

  • 二叉树

    ​ 二叉树的底层存储方式同样是讲数据存放到节点中，节点中包含上一个节点（父节点）的地址值，以及另外两个子节点的地址值。

    更多的数据结构以后再专门整理，主要目的是理解各个集合间的性能差异和原因。

  • 哈希表

    哈希表的篇幅较长也比较复杂，推荐一篇小白向的文章：一文彻底搞定哈希表，可以快速简单理解哈希表。

3.集合在Java中的实现

​ 集合一般都需要增、删、改、查等基本功能，为此Java在java.util包下提供了两个接口（也可以称为结合的根接口），分别为针对对象的集合接口：Collection以及针对键值对的集合接口：Map。这两个接口都分别有许多实现类，提供了多种集合的选择。

​ Collection家族中包含List系列集合和Set系列集合，主要用于存放单值元素。

2.Collection集合的遍历

  • 方式一：返回元素的数组再遍历这个数组

    在没有学习迭代器之前使用的方法，也可以当做一种思路。

  • 方式二：使用迭代器遍历

    因为每个集合的底层存储方式的不同，导致无法使用统一的方式访问集合内的元素。所有Java提供了一个迭代器接口：java.util.Iterator，里面定义了几个方法分别为：

    方法名 作用
    boolean hasNext() 判断是否有可迭代的元素（判断是否有下一个元素）
    <E> next() 返回迭代的下一个元素
    void remove() 移除当前迭代器指向的元素（这个是默认方法）

    不同的集合只需要在实现这个接口就可以解决不能统一访问方式的问题，实现方式一般是在集合类中定义一个内部类来实现这个接口，这也是设计模式之一的迭代器模式。

  • 方式三：使用forearm遍历

    可以使用一个特殊的for循环进行遍历，格式为for(数据类型 临时变量名:集合名)。使用forearm循环的前提是实现了java.lang.Iterable接口，这个接口中只有一个方法需要返回一个迭代器。所以forearm的实现其实也是依靠于迭代器的，只不过是使用上更加方便。

若只需要遍历集合建议使用forearm，若涉及到删除则建议使用方式二。

一般的List系列集合都可以使用索引操作，但实际性能根据底层存储方式的不同而不同。

List的常用实现类

  • Vector（向量）

    Vector底层数据结构为数组，默认初始容量为10，超出容量时扩容为原来的2倍，最大的特点是它是线程安全的。提供改变初始容量的构造器和增量的构造器。

  • ArrayList（动态数组）

    ArrayList底层数据结构为数组，默认初始容量为0，第一次使用时容量变为10，超出容量时扩容为原来的1.5倍，线程不安全。提供改变初始容量的构造器。

  • Stack（栈）

    Stack是 Vector的子类，它的最大特点是后进先出，可以理解为一个朝上开口的箱子，最后放进去的东西在箱子的最上面，想要拿到箱子底部的东西就必须把上面的东西拿开。但它的底层仍旧为数组。

    相比于Vector多提供了栈独有的操作方法：

    方法名 作用
    E peek() 查看栈顶对象
    E pop() 移除并返回栈顶对象
    E push（E item） 把对象放到栈顶（压栈），相当于add（E item）
    int search(Object o) 返回对象在栈中的位置，以1位基数
    boolean empty() 判断栈是否为空

  • LinkedList（双向链表）

    LinkedList的底层为双向链表，所以不推荐使用索引进行访问，是线程不安全的。除了操作双向链表的方法外，LinkedList还实现了java.util.Deque接口来支持用作双端队列、队列（先进先出）、单向链表、栈（后进先出）等各种数据结构，是一个十分强大的集合类。

4.Set系列集合

​ Set系列集合是部分有序、但都不可重复的，Set接口除了Collection提供的方法外没有额外提供方法。

Set的常用实现类

​ Set系列底层对应与不同的Map系列集合，想要了解底层实现可以先学习Map系列集合。

  • HashSet

    HashSet实际为HashMap，元素无序且唯一。保证元素唯一的方式是先比较存储对象的哈希值，若哈希值相同再通过equals()方法比较是否相等。

  • LinkedHashSet

    LinkedHashSet是HaseSet的子类,实际为LinkedHashMap，相比于父类增加了保证元素添加顺序的特性。

  • TreeSet

    TreeSet实际为TreeMap，元素按照大小排序且保证唯一。通过元素大小判断的方式来保证元素的唯一性，若元素大小相等则认为是相同元素。要求存储的元素必须实现java.lang.Comparable接口或在创建TreeSet 时指定一个java.util.Comparator接口。

2.Map集合的遍历

​ 由于Map接口并没有实现java.lang.Iterable接口所以不能直接使用foreach遍历

  • 方法一：调用Collection<V> values()方法再遍历返回的集合
  • 方法二：调用Set\<K> keySet()方法再遍历返回的集合，在遍历中通过V get(K key）获取value
  • 方法三：调用Set<Map.Entry<K,V>> entrySet()方法再遍历返回的集合

3.Map的常用实现类

​ Map系列的key均为唯一值。

  • Hashtable

    Hashtable是线程安全的，key和value不能为null，不保证元素插入顺序。

  • HashMap

    HashMap是线程不安全的，key和value可以为null。除此之外基本上与Hashtable一致，类似Vector与ArrayList的关系。Jdk1.8之后底层实现为数组+链表/红黑树。

  • LinkedHashMap

    LinkedHashMap为HashMap的子类，相比于父类增加了保证元素添加顺序的特性。

  • TreeMap

    键值对按照key的大小顺序进行排序。

  • Properties

    Properties是Hashtable的子类，它的key与value均为String类型，常用于存储配置属性。添加了一些方法，遇到时再进行讲解。
